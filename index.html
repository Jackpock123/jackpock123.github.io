Jekyll

I realised quickly that if I wanted blogging functionality for my website, having to hardcode every new post would be inefficient and messy when updating the website. I discovered Jekyll, a framework that generates static and blog-aware sites which let me host my website for free on GitHub.

Jekyll is written in Ruby, and allowed me to build my static website from dynamic components such as templates, liquid code and markdown. I made some basic sketches of the website and using the tutorials from cloudcannon and skills from the OneMonth bootcamp started to build my website from scratch.

During this time, I also learnt how to use Flexbox and CSS Grids on Scrimba (it's very cool that Per was part of FAC in 2015).

I'm a big fan of Jekyll, and in the process leading up to commiting my first build on GitHub in December 2018 I learnt:

  • To use liquid to build a blog which automatically updates content dynamically across several pages with each new post
  • Building and nesting templates for an efficient and functional website
  • Designing the layout, UX and UI for my own website
  • CSS Grids and Flexbox

JavaScript OneMonth Bootcamp

After completing the required JavaScript prerequisites on freeCodeCamp for F&C, I wanted to develop my understanding of JavaScript using a project based approach so in August 2019 I enrolled in the OneMonth JavaScript Bootcamp.

I built four applications: a Randomizer App, a Giphy App, a SoundCloud App and a data visualization exercise.

This Randomizer App that generates breaking news about the Royal Family by randomly selecting and concatenating data from 3 objects. Don't believe everything you read :)

  • Objects, Manipulating the DOM, variables, concatenate
  • Math.floor, Math.random, document.write

This Giphy App uses the search value to make AJAX requests to the giphy.com API and manipulates to DOM to show selected JSON data (the GIFs).

  • Functions and function scope, AJAX requests, parsing JSON data, build search bar and grab data for API call to giphy.com, passing variables as arguments, forEach loops
  • element.value, forEach(), element.innerHTML
  • .querySelector(), .addEventListener('keyup'), JSON.parse

This SoundCloud App uses the search value to make an API call to SoundCloud and then dynamically creates new divs which show selected information such as track titles and artwork. The user can then add these 'cards' to create their own playlist. The App stores the data locally to save the user's playlist and has a reset button to start over.

  • Manipulating the DOM to create 'cards' and display search results, build search bar and grab data for API call to soundcloud.com, passing variables as arguments, function scope
  • .createElement, .classList.add(), .appendChild(), .insertBefore(), localStorage.setItem(), localStorage.getItem(), localStorage.clear, location.reload()
  • forEach(), .querySelector, .addEventListener, SoundCloudAPI

This application displays rugby statistics using Google Charts, a Google Spreadsheet as a database to generate API data and Tabletop.js. This project also included tutorials on how to manipulate data with push, pop shift and reverse arrays and a comparison of the map() and forEach() functions.

  • Integrate data from Google Spreadsheets using Tabletop.js, displaying data, updating a database
  • .push(), pop(), shift(), reverse()
  • map(), forEach()
  • An introduction to 10 key concepts including objects, events, variables, functions, arrays and debugging
  • Creating a search bar, grabbing data and using it to make an AJAX call to an API (interacting with databases)
  • Parsing data with JSON and manipulating the DOM to show search results
  • Function scope
  • Displaying data using Google Charts and Tabletop.js
